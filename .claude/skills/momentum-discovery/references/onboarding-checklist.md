# Checklist de Onboarding — Cliente Nuevo

Fuente: `knowledge/03_TEMPLATES_Y_RECURSOS.md` seccion 1

## Informacion del Negocio
- [ ] Descripcion del producto/servicio
- [ ] Cliente ideal (perfil demografico/psicografico)
- [ ] Ticket promedio y rango de precios
- [ ] Ciclo de venta tipico
- [ ] Top 5 preguntas frecuentes
- [ ] Top 3 objeciones comunes
- [ ] Diferenciador vs competencia
- [ ] Accion deseada del bot (agendar, cotizar, vender, redirigir)
- [ ] Volumen actual de consultas

## Contenido para Entrenar el Bot
- [ ] Documentos de servicios/productos (PDFs, brochures)
- [ ] Precios actualizados (o rangos)
- [ ] Politicas relevantes (reembolsos, tiempos, restricciones)
- [ ] FAQs escritas
- [ ] Casos de exito o testimonios

## Accesos Tecnicos
- [ ] Canal: numero de WhatsApp + quien tiene SIM/acceso
- [ ] Canal: cuenta de Instagram/Facebook
- [ ] CRM actual (acceso) o decision de usar nuevo
- [ ] Sistema de citas (Calendly, etc)

## Operacion y Handoffs
- [ ] Quien recibe leads calificados (nombre, WhatsApp, Discord, email)
- [ ] Horario de atencion humana
- [ ] Que pasa fuera de horario
- [ ] Criterio de calificacion: que hace a un lead "bueno"
- [ ] Script de ventas actual (si existe)

## Tono y Personalidad
- [ ] Nombre del bot
- [ ] Tono: formal, semi-formal, casual
- [ ] Idioma o mezcla de idiomas
- [ ] Se hace pasar por humano o se identifica como bot

## Lo que MAS se suele escapar
1. Accesos al canal — el numero, quien lo tiene
2. Definicion clara del handoff — quien recibe y como
3. Templates de WhatsApp aprobados por Meta (si usan API oficial)
4. Diferencia entre lo que el bot puede prometer y lo que no
